# Тестовое задание
## Запуск
Клонируйте репозиторий.
```
git clone https://github.com/ArpolX/subscription-service
```
Используйте `docker compose up --build` для запуска сервиса. Он будет слушать стандартный `8080` порт.
## Небольшое описание
У нас есть 5 эндпоинтов, CRUDL-операции над подписками. Я учёл, что `user_id` не в моей зоне ответственности, поэтому стоит defauld на её генерацию. Также, если не указано `start_date`, оно будет выставлено по настоящему времени. Я предположил, что `end_date` может не быть, поэтому её указывать необязательно. 
На эндпоинте `/subscription/update` изменяется цена, дата начала и окончания подписки, по моему мнению это именно те параметры, которые стоит изменять в рамках одного пользователя и одной подписки
Более детально можно посмотреть по следующему пути:
```
http://localhost:8080/swagger/
```
