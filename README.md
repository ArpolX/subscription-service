# Тестовое задание
## Запуск
Клонируйте репозиторий.
```
git clone https://github.com/ArpolX/subscription-service
```
Используйте `docker compose up --build` для запуска сервиса. Он будет слушать стандартный `8080` порт.
## Небольшое описание
У нас есть **5 эндпоинтов**, CRUDL-операции над подписками.
### Особенности реализации
Я учёл, что `user_id` не в моей зоне ответственности, поэтому стоит default на её генерацию. Также, если не указано `start_date`, используется текущее время. Я предположил, что `end_date` является **необязательным**, поскольку подписка может быть открытой или её окончание неизвестно. 
Эндпоинт `/subscription/update` изменяет только следующие поля:
- `price`
- `start_date`
- `end_date`

Я считаю, что это те параметры, которые корректно обновлять в рамках одной подписки конкретного пользователя.
